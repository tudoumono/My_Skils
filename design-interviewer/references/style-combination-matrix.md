# style-combination-matrix.md — スタイル組み合わせマトリクス

## 概要

Phase 5 で使う。複数案は色違いではなくスタイルの組み合わせで分岐させる。単一スタイルより、2つを掛け合わせた方が「らしさ」が出て差別化できることが多い。ここでは代表的な10の組み合わせを、印象 / 向く用途 / 向く成果物 / カラー傾向 / タイポ傾向 / レイアウト傾向 / 強み / 弱み / 避けるべき使い方 の9項目で定義する。

## 判断基準（組み合わせの原則）

- **土台×アクセント**で考える。片方を「構造を決める土台」、もう片方を「印象を足すアクセント」にする。両方を主役にするとぶつかる。
- 情報量が多い成果物は、土台に静かなスタイル（Minimal/Corporate/Dashboard）を置き、アクセントを弱める。
- 差別化したい成果物は、土台は堅実にしてアクセントで攻める（無秩序を避けるため）。
- 3案では、A=堅実な組み合わせ、C=攻めた組み合わせ、B=中間、に散らす。
- 優先目的から逆算する: 信頼を優先するならAをCorporate系に寄せる。LP映えを優先するならBにBento/Editorial/Spatialを混ぜる。差別化を優先するならCでMap/Neo Brutalism/Luxury/Spatialを検討する。

---

## Minimal × Bento UI
- **印象**: 静かで整理された先進感。余白の中に機能が整然と並ぶ。
- **向く用途**: SaaSの機能紹介、プロダクト概要、モダンなコーポレート、個人開発の上質化。
- **向く成果物**: LPの機能セクション、サービスサイト、製品ダッシュボードの導入部。
- **カラー傾向**: 地は無彩色（白/オフホワイト）、アクセント1色。
- **タイポ傾向**: 太すぎないサンセリフ中心、見出しと本文の対比を明確に。
- **レイアウト傾向**: 12カラム上でカードの大小＝重要度。ガター一定、余白広め。
- **強み**: 一望性と上品さの両立。実装が比較的容易。**弱み**: 熱量・物語性は出しにくく、既視感が出やすい。
- **避けるべき使い方**: 全カード同サイズで等間隔に並べる（優先順位が消える）。

## Corporate × Flat Design
- **印象**: 堅実で分かりやすい、迷いのない安心感。
- **向く用途**: 大企業向け提案、公共・インフラ、金融/法務/医療、業務説明、社内承認。
- **向く成果物**: 提案スライド、サービスサイト、社内向け資料。
- **カラー傾向**: 落ち着いた基調色（Navy/Blue系）＋明快なアクセント1色、ベタ塗り。
- **タイポ傾向**: 可読性重視のサンセリフ、本文を小さくしすぎない、太字は要所のみ。
- **レイアウト傾向**: 明快なグリッド、左揃え、図解・比較表・実績ブロック、情報の段階を余白で。
- **強み**: 失敗しにくく、説明責任と信頼感が出る。**弱み**: 没個性・古臭くなりやすい。
- **避けるべき使い方**: ストック画像（握手・歯車）や汎用イラスト＋抽象コピーで埋める。

## Glassmorphism × Dark UI
- **印象**: 先進的で没入感のある、少し未来的な空気。デモ映えする。
- **向く用途**: AI/データ製品のヒーロー、技術デモ、ガジェット、映像・イベント演出。
- **向く成果物**: LPのファーストビュー、製品発表ページ、ピッチ画面、サイネージ。
- **カラー傾向**: ダークグレー地＋彩度高めのアクセント（Cyan/Teal系）、透過とGlowは1〜2箇所。
- **タイポ傾向**: シャープなサンセリフ、短い見出し、白は僅かに落とす、行間やや広め。
- **レイアウト傾向**: 中央に主題（デモ画面）、周囲に補助。テキスト少なめ、透過カードは要所限定。
- **強み**: 短時間で"先進"を印象づける。**弱み**: 可読性を落としやすく、安っぽいAI感に落ちやすい。全面採用で破綻。
- **避けるべき使い方**: 長文領域や高密度画面に低コントラストの透過を敷く。

## Editorial × Photo Focus
- **印象**: 雑誌的で情緒的、ブランドの世界観と温度が立つ。
- **向く用途**: ブランドストーリー、採用、事例、思想の共有。
- **向く成果物**: ストーリーLP、採用サイト、ブランドブック、スライド表紙。
- **カラー傾向**: 写真基準の抑えた配色（写真由来の中間色）、アクセントは最小。
- **タイポ傾向**: 見出しに個性ある書体を1つ、本文は端正に。行長を抑える。
- **レイアウト傾向**: 大きな写真＋余白＋引用、章立て。非対称も許容。
- **強み**: 共感・記憶に残る。**弱み**: 写真の質に全依存、機能訴求に弱い。
- **避けるべき使い方**: 低品質・文字化けAI画像を主役に置く。雰囲気写真で機能説明を置き換える。

## Neo Brutalism × Pop
- **印象**: 元気で大胆、既視感を壊す勢い。
- **向く用途**: Z世代向け、イベント、クリエイティブ、話題化狙い。
- **向く成果物**: キャンペーンLP、イベントサイト、SNS用ビジュアル、ポートフォリオ。
- **カラー傾向**: 高彩度2〜3色だが主役1色を決める、黒の太い枠線、明るい背景。
- **タイポ傾向**: 極太見出し、短文コピー、サイズの落差を大きく。
- **レイアウト傾向**: グリッド上で意図的にずらす、硬いシャドウ、強い境界線。
- **強み**: 圧倒的に目立つ・記憶に残る。**弱み**: 信頼・可読性・高級感を犠牲にしがち。
- **避けるべき使い方**: 金融/医療/公共、BtoB決裁者向け提案にそのまま使う。

## Luxury × Minimal
- **印象**: 静謐で上質、余白が語る高級感。
- **向く用途**: ハイエンド商材、プレミアムSaaS、上位プラン、招待制サービス。
- **向く成果物**: ブランドLP、上位プラン訴求、提案資料。
- **カラー傾向**: 深色地または Ivory/Warm Gray ＋ごく限定的な金/銅アクセント、彩度低め。
- **タイポ傾向**: 細めのセリフ/明朝を効かせ、字間を丁寧に。文字数少なめ、ウェイト抑制。
- **レイアウト傾向**: 大きな余白、少ないセクション、1画面1メッセージ、精密な配置。
- **強み**: 品格と信頼。価格の高さや専門性を支えやすい。**弱み**: 敷居が高く見える、機能訴求が弱い、情報量が多いと破綻。
- **避けるべき使い方**: 光沢・金縁の多用や暗背景だけで高級感を作る（成金化）。

## Corporate × Bento UI
- **印象**: 堅実さの上に一望性を足した、信頼できる情報整理。導入しやすい。
- **向く用途**: エンタープライズ/BtoB SaaS、業務改善、機能網羅の提案。
- **向く成果物**: サービスサイトの機能一覧、営業資料、提案資料の全体像スライド。
- **カラー傾向**: 落ち着いた基調（Navy/Slate）＋アクセント1色、カード地は白/微グレー、Success系は状態色として。
- **タイポ傾向**: 端正なサンセリフ、可読性優先、見出しで区画を明示。
- **レイアウト傾向**: 整然としたカード群（KPI/機能/事例）、大小で優先度、左揃え主体。
- **強み**: 情報量が多くても整って見える。複数機能を整理しやすい。**弱み**: 面白みに欠け、既存SaaSに似やすい。
- **避けるべき使い方**: カードを均一・過密に敷き詰め、すべての機能を同じ重要度で並べる。

## Bento UI × Map UI
- **印象**: 位置情報と各種情報を並置して一望させる、実用的な近未来感。
- **向く用途**: 位置ベースサービス、施設案内・来場者ナビ、地域サービス、店舗/物流管理、空間コンシェルジュ。
- **向く成果物**: ダッシュボード、案内アプリ画面、サービスLP、展示会デモ。
- **カラー傾向**: 地は無彩色、地図と各カードで役割色を区別（増やしすぎない）、ピンにアクセント色。
- **タイポ傾向**: サンセリフ、ラベルは小さくても読める太さで簡潔に、数値を読みやすく。
- **レイアウト傾向**: 地図を主区画に、周囲にBentoで状況説明・補助情報。
- **強み**: 空間×情報の同時把握。位置と状態を同時に伝えられる。**弱み**: 情報過多で地図が埋もれる。
- **避けるべき使い方**: 位置情報に意味のないサービスで装飾目的の地図を貼る（機能していない地図）。

## Minimal × Spatial UI
- **印象**: 静かな余白の中に奥行きが効いた、洗練された没入感。未来感を抑制して見せる。
- **向く用途**: 空間コンピューティング、体験デモ、先進プロダクト、複雑な関係性の可視化。
- **向く成果物**: 製品発表LP、体験紹介、ピッチ資料、展示の導入画面。
- **カラー傾向**: 無彩色地＋控えめなアクセント1色、影・層で奥行き。
- **タイポ傾向**: サンセリフ、短く言い切る、説明は段階的に、余白を活かす。
- **レイアウト傾向**: レイヤーと視差を情報階層に一致させる。奥行きある図解、装飾3Dにしない。
- **強み**: 上品な先進感。複雑な概念を上質に見せられる。**弱み**: 実装コスト高、酔わせるリスク。
- **避けるべき使い方**: 意味のない浮遊・回転・3D演出を主役にする。

## Dashboard UI × Corporate
- **印象**: 信頼できる分析画面、意思決定に耐える堅実さ。
- **向く用途**: 経営ダッシュボード、業務モニタリング、運用監視、レポート。
- **向く成果物**: 管理画面、分析画面、KPIレポート。
- **カラー傾向**: 無彩色地＋主役系列1色にアクセント、他はグレー。状態色は標準的に。
- **タイポ傾向**: 数値が読みやすい等幅寄り数字、ラベルは簡潔だが過度に小さくしない。
- **レイアウト傾向**: 最重要KPIを最上位→時系列→比較→詳細テーブルの順。グリッドで密度を制御。
- **強み**: 読み取りやすく信頼でき、実データに耐える。**弱み**: 地味・華が無い。余白不足で圧迫感が出やすい。
- **避けるべき使い方**: 円グラフ・二軸・虹色で"綺麗だが読めない"化、見栄えだけの意味のないグラフ（`data-viz-rules.md`）。

---

## 推奨パターン（3案への展開）

- **A案（Safe）**: Corporate × Flat Design、Dashboard UI × Corporate
- **B案（Attractive）**: Minimal × Bento UI、Editorial × Photo Focus
- **C案（Distinctive）**: Minimal × Spatial UI、Bento UI × Map UI、Luxury × Minimal

## NGパターン

- A/B/Cのレイアウトが同じで、カラーだけ変えて別案と言う。
- ターゲットに合わない攻め方をする（決裁者向けにNeo Brutalism等）。
- 実装コストの高い組み合わせを、コストを告げずに提案する。

## レビュー質問

1. 土台とアクセントの役割分担が明確か。両方主役になっていないか。
2. 成果物の情報量に対して、アクセントの強度は適切か。
3. 「避けるべき使い方」を踏んでいないか。
4. 3案に展開したとき、組み合わせの強度が散っているか。
5. この組み合わせはユーザーのどの不安を解消し、どの成果物で最も効果が出るか。
6. 既視感を避ける固有要素があるか。実装コストに見合う価値があるか。

## 出力例

> A案（Safe）は **Corporate × Flat Design**（堅実・実装容易、決裁者に安心を）、B案（Attractive）は **Minimal × Bento UI**（機能の一望性と上品さ）、C案（Distinctive）は **Glassmorphism × Dark UI**（先進感で差別化、透過はヒーロー1箇所限定）。C案のみ攻めていますが、可読領域は明基調に切り替えて破綻を防ぎます。
