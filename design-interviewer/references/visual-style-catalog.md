# visual-style-catalog.md — ビジュアルスタイル・カタログ

## 概要

Phase 4 で使う。19スタイルそれぞれについて「どんな時に選び、どんな時に避けるか」を判断できるようにまとめた辞書。各スタイルは次の6項目で記述する: 特徴 / 向く用途 / 向かない用途 / 相性の良い・悪いターゲット / 実装時の注意 / AIっぽく安く見える失敗例。スタイルは好みではなく、対象・目的・信頼の作り方・実装制約から選ぶ。単体スタイルだけで決めず、組み合わせは `style-combination-matrix.md` を併読する。

## 目次

Minimal / Bento UI / Corporate / Flat Design / Glassmorphism / Dark UI / Editorial / Neo Brutalism / Luxury / Pop / Spatial UI / Map UI / Dashboard UI / Apple風ミニマル / BtoB SaaS風 / 近未来AI風 / 和モダン / プレミアム / イベント・展示会向け

## 選定の判断基準（共通）

- 「目的（理解/行動/意思決定）」と「トーン（信頼/魅力/差別化）」の交点で選ぶ。好みで選ばない。
- 情報量が多い成果物ほど、装飾の弱いスタイル（Minimal / Dashboard UI / Corporate）が安全。
- 記憶に残したい・短時間で目を引きたいほど、強いスタイル（Neo Brutalism / Pop / Luxury）が効く。
- 目的別の起点: 信頼が必要なら Corporate / Minimal / Dashboard UI。初見の引きが必要なら Bento UI / Editorial / Spatial UI。差別化が必要なら Neo Brutalism / Map UI / Luxury。実装速度が重要なら Minimal / Corporate / Flat Design。
- 迷ったら A案=弱め、C案=強め、B案=中間、と3案に散らす材料にする。

## Minimal
- **特徴**: 余白・少ない色・整った字組みで内容だけを立たせる。装飾を足さず引き算で作る。
- **向く用途**: 信頼を求めるBtoB、上質さを出したいサービス紹介、情報を静かに読ませたいLP、落ち着いたトーンの採用ページ。
- **向かない用途**: 賑やかさや熱量が要るキャンペーン、要素を詰め込む必要がある高密度ダッシュボード。
- **相性の良いターゲット**: 決裁者、専門職、審美眼のあるユーザー。 **悪い**: 刺激と情報量、即時の楽しさを求める若年層向けキャンペーン。
- **実装時の注意**: 余白＝サボりではない。余白の大きさに一貫した基準（8px系）を持たせる。要素が少ない分、字間・行間・整列の粗が目立ち、精度が低いと単に寂しく見える。
- **AIっぽい失敗例**: 中央揃えの一言＋薄いグレー背景＋意味のない大きな余白だけで「ミニマル」と称する。情報設計が無く"スカスカ"に見える。

## Bento UI
- **特徴**: 大小のカードを弁当箱のように敷き詰め、要素ごとに区画を与える。機能一覧やダッシュボードで一望性が高い。
- **向く用途**: 機能紹介セクション、プロダクトの全体像提示、SaaSの特徴まとめ、ユースケース一覧。
- **向かない用途**: 物語性のある縦読みLP、長文の編集記事、1メッセージに集中させたい場面。
- **相性の良いターゲット**: 機能を比較検討する検討者、技術者、プロダクト理解を急ぐ人。 **悪い**: 情緒やブランド世界観で動く一般消費者。
- **実装時の注意**: カードの大小は「重要度」を表すために使う。情報の差をカードサイズの差に変換する。全部同じサイズだと情報の優先順位が消える。カード間ガターを一定に。
- **AIっぽい失敗例**: 同じサイズのカードをただ並べ、各カードに抽象アイコン＋一言だけ。中身が無く"それっぽい枠"だけになる。

## Corporate
- **特徴**: 落ち着いた配色、明快なグリッド、堅実な字組み。安心感と実績感を優先。
- **向く用途**: 大企業向け提案、IR、公共・インフラ、金融・法務・医療、社内承認資料など信頼が第一の商材。
- **向かない用途**: スタートアップの尖った差別化、若年層向けの熱量あるキャンペーン。
- **相性の良いターゲット**: 経営層、官公庁、管理部門、慎重な業界。 **悪い**: 新規性・意外性・遊びを評価するクリエイティブ層。
- **実装時の注意**: 「堅実」と「古臭い」は紙一重。余白と字組みを現代的にして鮮度を保つ。青一辺倒を避ける。固くなりすぎたら図解と余白で軽さを出す。
- **AIっぽい失敗例**: 握手・地球・歯車のストック画像＋濃紺グラデに抽象コピー。どの会社でも成立してしまう没個性。

## Flat Design
- **特徴**: 影・質感を排し、面と色で構成。軽く・速く・分かりやすい。説明性が高い。
- **向く用途**: 説明的なUI、手順・機能の図解、業務改善・教育系、幅広い層向けのサービス。
- **向かない用途**: 高級感・重厚感を出したい商材、写真の迫力が必要な案件。
- **相性の良いターゲット**: 一般ユーザー全般、初見ユーザー、ITリテラシー幅広い層。 **悪い**: プレミアム志向・尖ったブランドを期待する顧客。
- **実装時の注意**: 平面ゆえに階層が消えやすい。色・サイズ・余白で明確に優先順位をつける。色面とアイコンのトーンを統一する。
- **AIっぽい失敗例**: 原色の面をベタ塗りしただけでコントラスト設計が無く可読性が落ちる。汎用イラストを並べて独自性が消える。

## Glassmorphism
- **特徴**: すりガラス（半透明＋ぼかし）で奥行きを出す。背景が透ける軽やかさ。
- **向く用途**: 先進サービスのヒーロー、デモやイベントスクリーン、限定的なアクセント、モーダルやカードの一部。
- **向かない用途**: 長文の可読領域、情報密度の高い画面全体、アクセシビリティ最優先の業務UI。
- **相性の良いターゲット**: 新しさに反応する層、新規事業・AIデモ・展示会の来場者。 **悪い**: 可読性最優先の業務ユーザー、保守的な決裁者。
- **実装時の注意**: 透過は「1〜2箇所のアクセント」に留める。背景に十分なコントラストを確保し、文字が沈まないよう必ず検証する。
- **AIっぽい失敗例**: 画面全体を透過カードで埋め、背景の派手グラデが透けて文字が読めない。"それっぽいが使えない"典型。

## Dark UI
- **特徴**: 暗い基調で集中感・先進感を出す。発光や彩度の高いアクセントが映える。
- **向く用途**: 開発者ツール、AI/データ系プロダクト、分析画面、夜間利用、映像・ゲーム系の演出。
- **向かない用途**: 長時間の文書読解、明るさで安心感を出したい医療・教育・行政。
- **相性の良いターゲット**: 技術者、分析担当、ガジェット好き。 **悪い**: 高齢層を含む年齢層の広い一般ユーザー、明るい印象を好む消費者。
- **実装時の注意**: 純黒(#000)より少し明るいダークグレーを地にし、面の階層は Surface の明度差で分ける。テキストは純白でなく僅かに落とし、白文字を多用しすぎない。コントラスト比を確保。
- **AIっぽい失敗例**: 黒背景に発光する球体・パーティクルだけで中身が無い「AI感の演出」。

## Editorial
- **特徴**: 雑誌的な字組みと余白、写真・引用・見出しの強弱で読ませる。物語性が高い。
- **向く用途**: ブランドストーリー、事例記事、採用、思想を伝えたいサービス。
- **向かない用途**: 機能比較、操作UI、高密度ダッシュボード。
- **相性の良いターゲット**: 共感で動く読者、採用候補者、ファン層。 **悪い**: スペックを素早く比較したい購買担当。
- **実装時の注意**: 写真の質とコピーの編集がそのまま成果になる。字間・行間・行長を丁寧に。見出しに個性のある書体を1つ効かせる。
- **AIっぽい失敗例**: 低品質・文字化けしたAI生成写真や雰囲気写真だけで具体が無く、字組みも雑な"それっぽい雑誌風"止まり。

## Neo Brutalism
- **特徴**: 太い枠線・ベタ塗り・ずらしたレイアウト・硬いシャドウ。あえて洗練を崩して目立たせる。
- **向く用途**: 差別化したいスタートアップ、クリエイティブ、イベント、Z世代向け。
- **向かない用途**: 信頼・安心が第一の金融/医療/法務/公共、決裁者向け提案。
- **相性の良いターゲット**: 若年層、クリエイター、学生。 **悪い**: 安定感や品位を求める決裁者、保守的な業界、高齢層。
- **実装時の注意**: 「崩し」に一貫したルールを持たせる。無秩序と紙一重なので、グリッド自体は厳密に作り、その上で意図的に外す。
- **AIっぽい失敗例**: 蛍光色＋極太枠＋硬い影を脈絡なく多用し、ただ読みにくいだけになる。

## Luxury
- **特徴**: 深い色・低彩度・繊細な字組み・大きな余白・上質な写真。静けさと素材の質感で高級感を作る。
- **向く用途**: ハイエンド商材、プレミアムSaaS、ブランド、招待制サービス。
- **向かない用途**: 安さ・手軽さを訴求する商材、多機能SaaSの説明のような機能詰め込み画面。
- **相性の良いターゲット**: 富裕層、意思決定に品質を重視する経営者・上位顧客。 **悪い**: コスパ重視・機能比較重視の層。
- **実装時の注意**: 装飾でなく余白と素材品質で見せる。金・銅などのアクセントはごく限定的に。細めの明朝/セリフを効かせる。余白をケチらない。
- **AIっぽい失敗例**: 黒背景＋金グラデ＋光沢で"成金"感、あるいは暗いだけで読みにくい。上質さでなく過剰装飾になる。

## Pop
- **特徴**: 明るい多色・丸み・大きな文字・弾む動き。楽しさ・親しみ・勢い。
- **向く用途**: 一般消費者向けキャンペーン、エンタメ、教育、子ども/家族向け、個人開発、SNS施策。
- **向かない用途**: BtoB提案、規制業界など重い意思決定が絡む商材、高密度の業務UI。
- **相性の良いターゲット**: 若年層、ファミリー、初心者・カジュアル利用者。 **悪い**: 経営層、厳密さを求める専門職。
- **実装時の注意**: 多色でも「主役1色＋支える色」を決めて色数を絞る。CTAの優先度が装飾に埋もれないようにする。
- **AIっぽい失敗例**: 虹色を均等・無秩序に使い優先順位が消える。装飾過多で内容が埋もれる。

## Spatial UI
- **特徴**: 奥行き・レイヤー・視差で空間的に情報を配置。3D的な没入感。
- **向く用途**: 空間コンピューティング、体験デモ、展示、設備・ワークフローの空間表現、AR/VR文脈。
- **向かない用途**: 素早く読ませたいテキスト中心の資料、シンプルな業務UI、低スペック端末が前提の環境。
- **相性の良いターゲット**: 体験重視のユーザー、展示会来場者、技術デモの観客。 **悪い**: 効率・読みやすさ最優先の業務ユーザー。
- **実装時の注意**: 奥行きは情報の階層と一致させ、操作対象と装飾の差を明確にする。装飾の3Dにしない。`spatial-visualization-rules.md` を併読。
- **AIっぽい失敗例**: 意味のない浮遊カード・回転3Dオブジェクトで酔わせるだけ。

## Map UI
- **特徴**: 地図・位置情報を主軸に、ピン・ルート・エリア・レイヤーで情報を示す。
- **向く用途**: 位置ベースのサービス、店舗/施設案内、地域・イベント、物流・移動、来場者ナビ。
- **向かない用途**: 位置情報が価値に関係しない抽象サービス。
- **相性の良いターゲット**: 現地で行動するユーザー、運営者、来場者。 **悪い**: デスクで数値・抽象概念だけ見る層。
- **実装時の注意**: 地図は背景装飾ではなく意思決定の材料として使う。情報過多を避け目的の情報だけ強調。ピンの意味を色で区別しすぎない。`spatial-visualization-rules.md` を併読。
- **AIっぽい失敗例**: 装飾目的で地図画像（架空の地図含む）を雰囲気だけで貼り、位置情報が機能していない。

## Dashboard UI
- **特徴**: KPI・グラフ・テーブル・フィルタを密度高く配置し、状態を一望させる。
- **向く用途**: 分析画面、管理画面、業務モニタリング、意思決定支援。
- **向かない用途**: 感情訴求のLP、ブランドストーリー。
- **相性の良いターゲット**: 運用担当、マネージャー、分析担当。 **悪い**: 初見で軽く理解したい一般消費者。
- **実装時の注意**: レイアウトの前にデータの優先順位と状態設計（正常/異常/空）を決め、最重要指標を1つ最上位に。グラフは `data-viz-rules.md` に従う。密度と可読性の両立が肝。
- **AIっぽい失敗例**: 円グラフ・二軸・虹色や意味のない数字カードを多用し、綺麗だが何も読み取れない飾りダッシュボード。

## Apple風ミニマル
- **特徴**: 大きな余白、堂々とした写真、精密なタイポ、太めの短いコピー、控えめなモーション。Minimalの中でも"製品を主役にする"型。
- **向く用途**: プロダクトそのものが美しい商材、ハード/デバイス、新機能紹介、こだわりのSaaS。
- **向かない用途**: 情報量の多い比較表、業務密度の高い画面。
- **相性の良いターゲット**: 品質・プロダクト感度の高い消費者。 **悪い**: 価格や機能スペックをすぐ比較したい検討者。
- **実装時の注意**: 写真・プロダクト画像の質が全て。コピーは短く言い切る。余白と1画面1メッセージを徹底。Apple の模倣感を出さず、製品固有の価値を中心に置く。
- **AIっぽい失敗例**: 本物の製品画像が無いのに「Apple風」を真似て、中央大文字＋グラデーションだけの空虚な画面。

## BtoB SaaS風
- **特徴**: 明快なヒーロー、機能カード、導入ロゴ、数値効果、事例、セキュリティ訴求、料金表という定番導線。信頼と分かりやすさ重視。
- **向く用途**: 業務改善SaaSのLP・サービスサイト、営業資料。
- **向かない用途**: 一般消費者向けエンタメ、強い情緒ブランド。
- **相性の良いターゲット**: 現場担当〜決裁者、法人顧客。 **悪い**: 感情・エンタメ性で動く消費者。
- **実装時の注意**: 「よくあるSaaS」に埋もれやすい。3秒メッセージ・導入効果の数値・業界固有の実データや業務文脈で差をつける。アクセント色は1つ。
- **AIっぽい失敗例**: どのSaaSにも使える抽象イラスト・抽象ダッシュボード画像＋青系グラデ＋機能カード横並びで没個性。

## 近未来AI風
- **特徴**: 先進性・知性・精度を感じさせる。ダーク基調や余白の効いた明基調、限定的な発光アクセント、精密な図解。
- **向く用途**: AI/データ製品、R&Dデモ、技術発表・技術訴求。
- **向かない用途**: 温かみ・安心が主役の商材、保守的な業務システムの導入説明。
- **相性の良いターゲット**: 技術者、投資家、先進層、展示会来場者。 **悪い**: 安全性と運用性を第一に見る保守的な業界。
- **実装時の注意**: 「AI＝青紫グラデ」に逃げない。先進性は"余白・タイポの精度・データの見せ方"で出し、価値と仕組みを図解で具体化する。発光は1〜2箇所まで。
- **AIっぽい失敗例**: 青紫オーロラ背景＋光る球体＋ニューラルネット風の線＋「次世代」コピー。中身の無い"AIの記号"の羅列。

## 和モダン
- **特徴**: 余白（間）・縦組み・落ち着いた和色・自然素材の質感・繊細な線。静けさと品を日本文脈で表現。
- **向く用途**: 日本の伝統商材、和食・旅館・ホテル・工芸、地域ブランド、上質な国内向け採用。
- **向かない用途**: グローバルなテック、賑やかなキャンペーン、高速な業務ダッシュボード。
- **相性の良いターゲット**: 落ち着き・丁寧さ・地域性を評価する国内の品質重視層。 **悪い**: 派手なテック感・海外志向を期待する層。
- **実装時の注意**: 和色（藍・墨・生成り・朱）を役割で使う。和柄の貼り付けでなく、明朝と余白と素材で"間"を作る。
- **AIっぽい失敗例**: 桜・金・筆文字・和柄テクスチャを安易に重ねた"観光土産"風。品でなく安っぽさになる。

## プレミアム
- **特徴**: Luxuryを商用に寄せた型。上質だが機能も伝える。深色＋抑えたアクセント＋丁寧な字組みと明確な階層。
- **向く用途**: 高単価SaaS、エンタープライズ、コンサル・専門サービス、上位プラン訴求。
- **向かない用途**: 廉価・大衆向け、価格の安さ訴求。
- **相性の良いターゲット**: 予算のある法人、経営層・専門職、上位顧客。 **悪い**: 価格重視の個人、気軽さを求める初回ユーザー。
- **実装時の注意**: 「高そう＝敷居が高い」にしない。Luxuryより説明量を確保し、上質さと分かりやすさで信頼を作る。CTAは埋もれさせない。
- **AIっぽい失敗例**: 光沢・金縁・暗背景・明朝体を足しただけの"高級を装う安物"。

## イベント・展示会向け
- **特徴**: 遠くからでも読める大きな要素、強いコントラスト、短いコピー、記憶に残るモチーフ、動きや空間演出。短時間で目を引き足を止める。
- **向く用途**: 展示ブース、サイネージ、ピッチ、デモLP、来場者向け体験。
- **向かない用途**: じっくり読ませる文書・詳細仕様、細かい業務UI。
- **相性の良いターゲット**: 通りすがりの来場者、投資家、初見の顧客。 **悪い**: 落ち着いて詳細比較する購買担当。
- **実装時の注意**: 1メッセージを大きく、情報は絞る。視認距離を想定して最小文字サイズを大きく取り、「遠目・短時間・混雑環境で伝わるか」で検証する。
- **AIっぽい失敗例**: 情報を詰め込みすぎて遠くから読めない。派手な背景だけで何のサービスか分からない。

## 推奨の組み合わせパターン

## NGパターン

- スタイル名の響きだけで決める。ターゲットとCTAを無視して流行を選ぶ。
- 写真や図解の実体なしで雰囲気だけ作る。
- 3案すべてに同じレイアウトを使い、色だけ変えて「別案」と称する。

## レビュー質問（スタイル選定後に自分へ）

1. 選んだスタイルは「目的×トーン」から導けているか。ユーザーの好みに引きずられていないか。
2. このスタイルは「誰の不安を減らし、CTAへの導線を強めるか」を説明できるか。
3. その成果物の情報量に、このスタイルの装飾強度は合っているか。実装制約（工数・端末・環境）に無理はないか。
4. 「向かない用途」に当てはまっていないか。「AIっぽい失敗例」を1つでも踏んでいないか。
5. 3案に展開したとき、スタイルの強度が Safe→Attractive→Distinctive で散っており、違いを色以外で説明できるか。

## 出力例（スタイル提案の書き方）

> このLPは「信頼を得て導入判断につなげる」目的なので、A案（Safe）は **Corporate × Flat Design**（安心・実装容易）、B案（Attractive）は **BtoB SaaS風 × Bento UI**（魅力と一望性）、C案（Distinctive）は **近未来AI風（明基調）× Editorial**（差別化）を軸に提案します。決裁者には信頼材料が要り、LPでは機能の理解速度が効くため、いずれも青紫グラデには逃げず、先進性は余白と字組みで出します。
