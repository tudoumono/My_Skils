# typography-guide.md — タイポグラフィガイド（日本語UI前提）

## 概要

Phase 8 で使う。日本語UIは英語UIのサイズ感をそのまま持ち込むと破綻する。ここでは日本語の可読性を守るための基準をまとめる。文字設計に入る前に必ず読む。

- **向いている用途**: LP・アプリ画面・ダッシュボード・スライドの文字設計、デザイントークン、日本語中心のUIレビュー。
- **向いていない用途**: 装飾書体だけでブランド感を作る、英語の見本をそのまま日本語に適用する。

## 中心思想

- 日本語は文字が複雑（漢字の画数）で、英語より**大きめ・行間広め**が読みやすい。
- 英語LPの「小さい本文＋詰めた行間」をそのまま日本語に適用しない。
- 太字を多用すると強調が効かなくなる。強調は"1画面に少数"。

## 扱う要素

Font Family / Type Scale / Font Weight / Line Height / Letter Spacing / Text Width / 日英の違い / 用途別サイズ（LP / Webアプリ / スライド）。

## Font Family

- **和文サンセリフ（ゴシック）**: 汎用UI・BtoB・SaaS・ダッシュボードの基本。可読性が高い。例: Noto Sans JP、Hiragino Sans、Yu Gothic UI、system-ui。
- **和文セリフ（明朝）**: 上質・和モダン・エディトリアル・プレミアムの見出しや本文に。品が出る。ただし細すぎる明朝は避け、本文は可読性優先。
- **欧文サンセリフ**: 数字・英字・ロゴに。Inter等を和文と組むときはウェイトと字面を合わせる。
- 判断: 信頼/実務=ゴシック（例: Noto Sans JP＋Inter併用）、上質/情緒=明朝を効かせる。和欧混植は「和文を主、欧文を従」で選ぶ。

## Type Scale（比率で作る）

- スケールは比率で刻む（例: 1.125〜1.25倍）。バラバラのサイズを感覚で置かない。
- 例（本文16px基準・LP想定）: 本文16 / 小14 / 見出し小20 / 中24 / 大32 / 特大40〜48。
- 見出しと本文の**サイズ差を明確に**（隣接段階でなく1つ飛ばす等）。差が曖昧だと階層が消える。
- 日本語のHeroコピーは英語LPのように無理に巨大化しない。長いなら短く分割してから大きくする。

## Font Weight

- 本文は Regular(400)〜Medium(500)、UIラベルは 500〜600、見出しに 600〜700。
- **太字を多用しない**。1画面で太字が多いと、どれも目立たず強調が死ぬ。強調箇所だけに限定する。
- 細すぎるウェイト（Thin/Light）を小さい本文に使わない（日本語は潰れる）。

## Line Height（行間）

- 日本語本文は **1.7〜2.0** を基本に、やや広めにする（英語の1.4〜1.5より広い）。
- 見出しは詰めてよい（1.2〜1.4、大見出しは1.1〜1.3）。長い見出しは折り返し位置に注意。
- UIラベルは 1.3〜1.5、スライド本文は 1.35〜1.6 が目安。
- 行間が詰まると漢字の塊が読みにくくなる。広すぎるとまとまりが崩れる。情報密度を稼ぐために行間を詰めない。

## Letter Spacing（字間）

- 和文本文は基本 0（ベタ組み）〜わずかに広げる程度。詰めすぎない。
- **負の字間（トラッキング詰め）を日本語に使わない**。欧文見出しの流儀を持ち込まない。
- 見出し・全大文字の欧文、小さな英字ラベルはやや広げると締まる。
- 明朝の大見出しは僅かに広げると上品になる。

## Text Width（行長）

- 日本語本文の1行は **全角30〜40字程度**（最大でも45字）に抑える。広げすぎると視線が戻れず読みにくい。
- LPやコンテンツは max-width を設けて長文を横に伸ばさず、段落間に余白を入れる。
- カード内の短文はこの限りではないが、段落が続く箇所は必ず幅を制御する。
- ダッシュボードでは短いラベルと数値を優先し、長い説明はツールチップや詳細画面に逃がす。

## 日本語と英語の違い（要点）

- 同じ級数でも日本語は英語より詰まって見える → 大きめ・行間広めが基本。
- 英語は単語間スペースで区切れるが、日本語は連続する → 行間と行長で読みやすさを作る。
- 英語LPの「12〜14pxの本文」をそのまま日本語にしない（14pxは日本語だと小さい）。
- 欧文の全大文字強調は日本語にそのまま効かない → 太さ・サイズ・色で強調する。
- 和欧混在時は高さ（ベースライン・字面）が揃うフォントの組み合わせを選ぶ。

## 用途別サイズ目安

### LP向け（PC想定、SPは一回り調整）
- 本文: 16〜18px / 行間 1.8前後 / リード文: 18〜22px
- 小見出し: 20〜24px / 見出し: 28〜36px / Hero大見出し: 40〜56px
- キャプション・法務注記: 14px以上。例外でも可読性を確認する。
- Heroコピーは短く大きく。本文は詰め込まない。

### Webアプリ / 管理画面向け
- 本文/ラベル/テーブル: 14〜16px（情報密度優先だが、日本語本文と操作ラベルは14px未満にしない。補助メタ情報のみ13pxを例外検討可）
- ページタイトル: 20〜28px / セクション見出し: 16〜20px
- 数値KPIの大表示: 24〜40px。桁を揃え、等幅数字が使えると読みやすい。
- 行間はやや詰めてよい（1.5前後）。

### スライド資料向け（投影想定）
- 本文: 18〜24pt（後方席から読める大きさ）
- 見出し: 28〜40pt / 表紙タイトル: 40〜60pt
- 1スライドの文字数を絞る。小さい注記でも16pt以上。投影環境では小さくしすぎない。

## 日本語UIで守ること（チェックリスト）

1. 本文を小さくしすぎない（LP16px以上、アプリ14px以上、スライド18pt以上）。
2. 行間をやや広めに（本文1.7〜2.0）。
3. 長文の横幅を広げすぎない（全角30〜40字）。
4. 見出しと本文のサイズ差を明確に。
5. 太字を多用しない（強調は少数に）。
6. 英語LPのサイズ感・負の字間を日本語にそのまま持ち込まない。
7. 細ウェイトを小さい本文に使わない。

## NGパターン

- 英語テンプレ由来の13px本文＋行間1.4を日本語に流用し、漢字が潰れて読みにくい。
- 日本語本文や操作ラベルが14px未満。
- 見出しと本文が2px差で、階層が視覚的に伝わらない。
- 全部Boldで、重要箇所が沈む。
- 本文が画面幅いっぱいに伸び、1行が長すぎて読み疲れる。
- 日本語の長いHeroコピーを英語LPのように巨大化し、折り返しが破綻する。
- 字間を負にして詰まった印象にする。

## レビュー質問

1. 本文サイズは用途の下限を満たしているか。
2. 行間は日本語向けに広めか。詰めて密度を稼いでいないか。
3. 行長（横幅）は制御されているか。長文は読み切れる幅か。
4. 見出しと本文のサイズ差は一目で分かるか。スマホ幅でも見出しが破綻しないか。
5. 太字・強調が多すぎないか。
6. 英語基準のサイズ・字間をそのまま使っていないか。
7. 数字・単位・注釈は読み間違えにくいか。CTAの文字サイズはボタン内で余裕があるか。
8. 日本語と英数字が混在したとき高さが揃っているか。

## 出力例

> タイポは日本語UI前提で設計します。Font FamilyはNoto Sans JP＋Inter（数字）。本文16px/行間1.75、本文の行長は max-width で全角約38字に制限。見出しは32px・行間1.3・Bold、Heroのみ48pxで短く言い切ります。UIラベルは14px/600。強調は各セクション1〜2箇所のMediumに留め、太字の乱用は避けます。
